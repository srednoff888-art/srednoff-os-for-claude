# QUALITY_GATE.md — чеклист перед завершением задачи

> **Источник истины**: `.claude/rules/40-quality-gate.md` (краткая версия того же списка). Этот файл — более подробный чеклист-раскрытие того же набора проверок, для ручного использования. При конфликте выигрывает `rules/`; при правке одного — проверь и обнови другой (найдено архитектурным ревью 01.07.2026 как риск рассинхронизации трёх копий одного и того же: `CLAUDE.md`/`.claude/rules/`/`.agent/`).

Перед финальным ответом Claude обязан пройти этот чеклист.

## 1. Scope
- [ ] Задача понята.
- [ ] Нетривиальные assumptions записаны.
- [ ] Если нужны были вопросы — они заданы.
- [ ] Сделано ровно то, что просили, без лишнего расширения scope.

## 2. GitHub / docs
- [ ] GitHub проверен, если задача нетривиальная.
- [ ] Найдены и сравнены похожие решения.
- [ ] Официальная документация проверена для критичных API/SDK.
- [ ] Лицензии учтены, если использовались идеи из OSS.

## 3. Code quality
- [ ] Код минимален и понятен.
- [ ] Нет hardcoded secrets.
- [ ] Нет мёртвого кода.
- [ ] Ошибки обрабатываются.
- [ ] Типы корректны.
- [ ] Публичные API не сломаны без причины.
- [ ] Документация обновлена, если поведение изменилось.

## 4. Tests
- [ ] Unit tests добавлены/обновлены, если применимо.
- [ ] Integration tests добавлены/обновлены, если применимо.
- [ ] Regression test добавлен для багфикса, если возможно.
- [ ] Ручной тестовый сценарий описан.

## 5. Commands
Выполни доступные команды:

```bash
npm run lint
npm run typecheck
npm test
npm run build
```

Если проект не Node.js, выбери аналоги.

## 6. Security
- [ ] Проверены auth boundaries.
- [ ] Проверены permissions/RLS.
- [ ] Проверен ввод пользователя.
- [ ] Нет утечки PII/secrets.
- [ ] Нет опасных SQL/command injections.
- [ ] Нет destructive действий без подтверждения.

## 7. Deploy readiness
- [ ] Env vars перечислены.
- [ ] Migration plan есть.
- [ ] Rollback plan есть для рискованных изменений.
- [ ] Логи/мониторинг понятны.
- [ ] Production impact описан.
