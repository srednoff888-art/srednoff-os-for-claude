# USER_BRIEFING.md — как опрашивать пользователя

Claude должен получать вводные быстро и точно.

## Правило 5 вопросов

Если задача неясна, задай максимум 5 вопросов.

Плохие вопросы: "Расскажите подробнее", "Что вы хотите?", "Какие пожелания?"

Хорошие вопросы:
- "Деплой делаем на Vercel или локально?"
- "База Supabase уже создана или нужно создать миграции с нуля?"
- "Telegram bot token уже есть?"
- "Нужна авторизация через email, Google или Telegram?"
- "Можно добавлять новые npm-зависимости?"

## Если пользователь не отвечает

Не останавливайся бесконечно. Сделай assumptions:

```md
Assumptions:
- Деплой: Vercel
- БД: Supabase Postgres
- Frontend: Next.js
- Auth: Supabase Auth
- Language: TypeScript
```

## Уровни подтверждения

### Не нужно подтверждение
Чтение файлов; локальный анализ; запуск тестов; создание черновика; изменение dev-only файлов.

### Нужно подтверждение
Установка новых production dependencies; изменение схемы БД; migration, влияющая на данные; изменение env vars; деплой в production; отправка email; удаление файлов; массовое изменение кода.

### Запрещено без явного подтверждения
Удаление production данных; отключение security; публикация секретов; платные операции; изменение billing/domain/DNS.
