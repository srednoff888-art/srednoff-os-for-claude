# 60-exec-plans.md

> Канонический источник. Развёрнутая версия — `.agent/PLANS.md`; при правке одного файла проверь и обнови второй.

Создавай ExecPlan для больших задач:

- архитектура; БД; auth; payments; deploy;
- несколько модулей; высокий риск; сложная интеграция.

Формат:

```md
# ExecPlan: <name>

## Goal
## User value
## Current state
## Assumptions
## Questions
## GitHub research
## Official docs checked
## Architecture
## Steps
## Data model / migrations
## API contracts
## Security model
## Testing plan
## Rollback plan
## Definition of done
## Progress log
```

## Если сам агент (не сабагент) упирается в блокер или сбой

(Найдено архитектурным ревью 01.07.2026: `90-subagent-contract.md` требует явный disposition только от ДЕЛЕГИРОВАННЫХ вызовов — для собственного сбоя primary-агента правила не было.)

- Build/test/deploy шаг падает → не повторяй молча ту же команду; зафиксируй в Progress log точную ошибку, диагностируй причину, и либо чини, либо явно отметь `## Blocked` с тем, что нужно от пользователя.
- Инструмент недоступен / внешняя зависимость сломана → сообщи об этом прямо, не притворяйся, что всё ок, не додумывай результат.
- Задача частично сделана и нельзя продолжить в этом ответе → финальное сообщение должно явно называть: что сделано, что не доделано, что нужно для продолжения (тот же принцип disposition, что и для сабагентов: `done`/`blocked`/`deferred`/`failed`, не обрывать молча).
