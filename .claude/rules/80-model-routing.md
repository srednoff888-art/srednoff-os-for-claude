# 80-model-routing.md

Подчинено **Принципу №1** (качество первично; экономия — при равном качестве; см. `70-skills-registry.md`).

Выбор модели = выбор **минимально достаточной мощности для НУЖНОГО качества — и не ниже**.

| Тип работы | Модель | Почему |
|---|---|---|
| Детерминированное/механическое: формат, конвертация, генерация по шаблону, простой grep/правка, чтение, lint, короткие ответы | **Haiku** | рассуждение не нужно → дёшево без потери качества |
| Экспертная работа по домену: код-ревью диффа, реализация фичи, отладка, тесты, рефактор, средний анализ | **Sonnet** | баланс качества/стоимости для основной инженерной работы |
| Сложное рассуждение: архитектура, многошаговый план, полный аудит безопасности, deep research, мульти-агент-оркестрация, тонкий баг, проектирование схемы/контрактов | **Opus** (Fast — для скорости) | задача требует глубокого рассуждения → не занижать |

## Правила
- **НЕ занижай модель**, если задача требует рассуждения, ради экономии. Ошибка из-за слабой модели дороже сэкономленных токенов.
- Можно **поднять** модель на критичном шаге даже внутри «дешёвой» задачи (напр. финальное ревью архитектуры) и опустить обратно.
- Длинные механические прогоны (массовая правка по шаблону) — Haiku оправдан.
- Соответствие группам реестра: **G1≈Haiku, G2≈Sonnet, G3≈Opus** — ориентир, не жёсткое правило: решает требуемое качество.
- Переключение в Claude Code — `/model` или per-agent `model` в подзадаче. Нетривиальный выбор зафиксируй 1 строкой.
