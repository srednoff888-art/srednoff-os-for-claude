# 90-subagent-contract.md

Контракт делегирования сабагентам — чтобы делегирование НЕ раздувало контекст.

При вызове сабагента (Agent/Task):
- В промте задай: **цель, границы, нужный формат, лимит вывода** (напр. «верни ≤ N строк: только итог + файлы:строки, без пошагового лога»).
- Сабагент возвращает **результат, а не транскрипт**: вывод/находки/решение. Родитель пересказывает пользователю только важное (relay-what-matters), не дамп.
- Один сабагент = одна чёткая задача. Несколько независимых — параллельно одним сообщением.
- Read-only разведка по многим файлам → агент `Explore`; план реализации → агент `Plan`. Не плодить агентов без нужды.
- Сабагент наследует **Принцип №1** (качество первично) и роутинг моделей (`80-model-routing.md`): мощность под требуемое качество.
- Внешних/swarm-агентов звать только после верификационного гейта (`70-skills-registry.md`).
- **Финальный disposition обязателен** (концепция адаптирована из paperclipai/paperclip, MIT — их паттерн: агент обязан завершить тикет одним из фиксированных терминальных статусов перед выходом, не обрываться молча). Каждый background/delegated вызов (Agent, Task, фоновый Bash) должен закончиться явным исходом, который родитель фиксирует в ответе пользователю: `done` (сделано, результат ниже) / `blocked` (нужен ввод/решение пользователя — указать что именно) / `deferred` (отложено, указать почему) / `failed` (не получилось — указать причину). Не оставлять читателя гадать, доделалось ли что-то в фоне.
